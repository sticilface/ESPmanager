Rules for HTTP JSON manifest file

I've had a look and I agree that the link doesn't work. I'll make an issue and fix that. The structure is a JSON object.  saveto is the FS filename, must have the preceding /, location is the relative path to the manifest file that the file can be found, md5 is the md5sum of the new file. This is used to compare against the current file to avoid downloading it if it is unchanged. The new binary must be called firmware.bin and save to = "sketch". Finally the keys filecount tell the updater how many to expect, and overwrite means the md5 is ignored and all files are written. there is an isurl option for each file as well, that can be used to specify that the location is a whole url, and not a relative one. Allows you to download external resources, otherwise the link is treated as relative.
